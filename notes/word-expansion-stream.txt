word expansion stream

Quote removal is specified to come last, but only removes quotes that were present in the original word.
Therefore we can remove them early and store "quoted" status of each character with an internal escape
or out-of-line.

expand
	- needs buffer or lookahead for names -> directly access the input string
	- emit expand_start, expand_end special chars
	- emit escape special if special char appears in input
split
	- only splits on chars from expansion -> needs to know state
pathname
	- only quotes in original can disable wildcards -> needs to know state
	- may be inefficient if pattern not stored -> storage needs to know state too
quoteremove
	- doesn't happen on pathname matches
	- only removes quotes from original -> needs to know state
